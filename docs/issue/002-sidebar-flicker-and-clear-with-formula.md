## 现象/背景

用户反馈在固定侧边栏查看流式翻译时，内容刚输出几行中文会先闪烁一下，随后整块翻译内容被清空并不再继续显示。当前经验上“原文/译文包含公式”时更容易复现。

## 复现步骤

1. 打开一篇包含公式的 PDF（样本文档：`<TBD>`）。
2. 启动翻译并在固定侧边栏观察流式输出（入口：`<TBD>`）。
3. 在输出早期阶段观察是否出现一次闪烁。
4. 闪烁后翻译内容被清空，后续不再显示。

- 需要用户补充：
  - 最小稳定复现文档/页码
  - 具体操作路径（是否涉及切换标签页/滚动/重新触发翻译）
  - 复现概率与是否与公式密度相关

## 预期结果 / 实际结果

- 预期：固定侧边栏在流式输出期间稳定渲染，不闪烁、不清空，并持续显示后续译文。
- 实际：含公式场景下有概率在输出前几行后闪烁并清空，导致后续内容不可见。

## 根因分析

已确认根因（2026-02-12）：

- `math-textbox` 在每次流式 chunk 更新时都会先 `remove()` 旧 overlay，再重建并 `innerHTML = render...`。
- 当内容包含公式且 chunk 高频刷新时，会出现可见“先移除后重建”的闪烁。
- 一旦重建链路中任一步骤抛错（渲染/样式同步/DOM 更新），旧 overlay 已被移除，但 `overlay-visible` 状态仍可能保持为 true，导致底层 `editable-text` 被隐藏且无可见内容，表现为“闪烁后清空并停留空白”。

修复思路：改为“复用 overlay + 就地更新内容 + 失败保留上一帧”，避免中间空窗与内容丢失。
